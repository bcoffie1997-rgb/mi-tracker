# Cold-email setup for `trymindyai.com`

Goal: stand up `branden@trymindyai.com` on Google Workspace, configure DNS so
the inbox can both receive and authenticate outbound mail, warm the domain so
it doesn't get blacklisted on day one, then start cold outreach to the 219
contacts in the tracker.

**Estimated time**: 30–45 min hands-on setup today + 14 days passive warmup
before the first cold send.

---

## Pre-flight checklist

Before you start, have:

- [ ] **Credit card** for Google Workspace ($6/user/month, billed monthly)
- [ ] **Namecheap login** — domain control panel for `trymindyai.com` (already at
  `ap.www.namecheap.com/domains/domaincontrolpanel/trymindyai.com/domain`)
- [ ] **A phone** for Workspace verification SMS
- [ ] **A recovery email** that is NOT `@trymindyai.com` (use your gmail)
- [ ] ~45 min uninterrupted

## Phase 1 — Google Workspace signup (≈15 min)

1. Go to **<https://workspace.google.com/business/signup/welcome>**
2. Pick **Business Starter** — $6/user/month, includes a 30GB Gmail inbox at
   your domain. Don't pay for Standard yet; you only need the inbox.
3. When asked for your domain, enter `trymindyai.com` (NOT `www.trymindyai.com`).
4. Create your primary user: `branden@trymindyai.com`. Use a strong unique
   password and save it to your password manager.
5. When Google asks "How will you verify the domain?" → it'll offer a TXT
   record. Note the value (`google-site-verification=...`) — you'll paste it
   into Namecheap in Phase 2.
6. **Stop here** — don't click "Verify" yet. Add the DNS records first, then
   come back and verify.

> [!NOTE]
> If you want a second mailbox later (`eric@trymindyai.com`), add it via
> Workspace Admin → Users → Add user. Each user is +$6/mo. For warmup, **one
> mailbox is enough** — don't add the second until that one is warm.

## Phase 2 — DNS records at Namecheap (≈10 min)

In the Namecheap tab you have open, click the **Advanced DNS** tab (top of the
domain detail card, fourth tab).

### Records to add

Add each record below by clicking **ADD NEW RECORD**. Set TTL to "Automatic"
unless specified.

#### 2.1 — Domain verification

| Type | Host | Value | TTL |
|------|------|-------|-----|
| TXT  | `@`  | `google-site-verification=<the-token-from-Phase-1-step-5>` | Automatic |

#### 2.2 — MX records (Gmail receiving)

Google now uses a single MX endpoint. Add ONE record:

| Type | Host | Value | Priority | TTL |
|------|------|-------|----------|-----|
| MX   | `@`  | `smtp.google.com` | 1 | Automatic |

> [!IMPORTANT]
> If Google's signup wizard still shows the legacy 5-record format
> (`aspmx.l.google.com`, `alt1.aspmx.l.google.com`, etc), use exactly what the
> wizard provides — copy/paste each one with the priority Google lists. Both
> formats currently work; new accounts get the single-record version.

#### 2.3 — SPF (TXT, authorizes Google to send as you)

| Type | Host | Value | TTL |
|------|------|-------|-----|
| TXT  | `@`  | `v=spf1 include:_spf.google.com ~all` | Automatic |

> [!WARNING]
> You can only have **one** SPF record on a domain. If Namecheap already shows
> a TXT record starting with `v=spf1`, **don't add a second** — edit the
> existing one to merge includes if needed.

#### 2.4 — DKIM (TXT, cryptographic signature)

You can't generate this value until *after* Google verifies your domain. So:

- [ ] **Add this record blank for now** — skip and come back to step 2.4 after
  Phase 3 verification completes.
- [ ] Once verified: Workspace Admin → **Apps → Google Workspace → Gmail →
  Authenticate email** → click **Generate new record** → copy the TXT value
  and the host (`google._domainkey`) and add at Namecheap.

| Type | Host | Value | TTL |
|------|------|-------|-----|
| TXT  | `google._domainkey` | *(generated by Google after verification)* | Automatic |

#### 2.5 — DMARC (TXT, monitoring policy)

| Type | Host | Value | TTL |
|------|------|-------|-----|
| TXT  | `_dmarc` | `v=DMARC1; p=none; rua=mailto:branden@trymindyai.com; pct=100` | Automatic |

> [!NOTE]
> We start at `p=none` (monitor only) so you get the daily aggregate reports
> without rejecting any real mail. After 30 days of clean reports, tighten to
> `p=quarantine`, then eventually `p=reject`. Don't skip `none` — going
> straight to `reject` on a new domain WILL drop legitimate mail.

#### 2.6 — Save and wait for propagation

After saving all records, propagation typically takes 15 minutes but can take
up to a few hours. While you wait, check propagation status:

```bash
dig +short txt trymindyai.com
dig +short mx trymindyai.com
dig +short txt _dmarc.trymindyai.com
```

(Or use the web tool: <https://mxtoolbox.com/SuperTool.aspx>)

## Phase 3 — Verify domain in Google Workspace (≈5 min)

1. Return to the Workspace setup wizard (the tab you left in Phase 1).
2. Click **Verify my domain**.
3. If the TXT record has propagated, you'll see a green check. If not, wait
   10 min and retry.
4. Once verified, Google activates the inbox. You can now log into
   <https://mail.google.com> with `branden@trymindyai.com`.
5. **Go back to step 2.4** and add the DKIM TXT record now.

### Confirm everything authenticates

Send a test message from `branden@trymindyai.com` to a personal Gmail
address. Open the received mail → click the three-dot menu → **Show original**.
You should see:

```
SPF: PASS with IP ...
DKIM: 'PASS' with domain trymindyai.com
DMARC: 'PASS'
```

If any of these say `FAIL` or `NEUTRAL`, **stop and fix before continuing**.
Cold-sending with broken auth is worse than not sending at all.

## Phase 4 — Warmup (14 days, passive)

A brand-new domain that suddenly fires off cold outreach gets flagged within
hours. Mailbox providers (Gmail, Outlook) need to see consistent low-volume
healthy sending before they trust you.

### Pick one warmup approach

#### Option A — Automated warmup tool (recommended)

| Tool | Cost/mo | Notes |
|---|---|---|
| **Mailwarm** | $79 | Simple, established. Auto-replies between warmed inboxes. |
| **Warmup Inbox** | $39 (single inbox) | Cheapest viable option. |
| **Lemwarm (Lemlist)** | $29 standalone | Solid, slightly limited free reach. |
| **Instantly warmup** | $30 (cheapest plan) | Bundled if you ever upgrade to their sequencer. |

All four work the same way: they send small low-key emails from your inbox to
thousands of other warmed inboxes worldwide, each of which auto-opens, replies,
and marks-as-important. Your inbox builds a positive reputation gradient.

**Setup**: sign up, paste the IMAP/SMTP credentials Google gives you (you'll
need to enable 2FA on the Google account first, then create an app password
under <https://myaccount.google.com/apppasswords>).

Run it for **14 days minimum** before sending any cold outreach.

#### Option B — Manual warmup (free, slower)

- [ ] Day 1–3: send 3–5 emails/day to actual contacts (friends, prior
  customers) and have them reply naturally
- [ ] Day 4–7: 8–10/day, still all replies expected
- [ ] Day 8–14: 15–20/day, mix in newer recipients
- [ ] Throughout: open and reply to messages people send you; star important
  ones; mark spam if anything looks shady

Option B works but is fragile (one missed day → start over). Option A is the
safer bet.

### Things to do during the warmup window

- [ ] Set a real email signature with full name, title, company, physical
  address, and unsubscribe instructions (CAN-SPAM requirement)
- [ ] Set a friendly profile picture (Gmail account → personal info)
- [ ] Send/receive a few "thank you" replies to anyone you mailed
- [ ] In Postmaster Tools (<https://postmaster.google.com>), register
  `trymindyai.com` and start watching reputation — green is what you want by
  day 10

## Phase 5 — First cold send (Day 15+)

After 14 days of warmup, you can start cold outreach. Two paths:

### Path A — Use the tracker's "Open in Gmail" flow (available today)

From `/outreach` in the tracker:

1. **Queue tab** → find a contact who's due → click **Draft**
2. The Compose modal opens with the merged template
3. Edit anything that feels canned/generic — make each send feel personal
4. Click **Open in Gmail** → Gmail opens with the message pre-filled, sending
   from `branden@trymindyai.com`
5. Send from Gmail directly. Come back to the tracker, click **Log as sent**

### Path B — Send via API from the dashboard (needs the OAuth route I haven't built yet)

Once you're confident in the manual flow, ping me and I'll wire up the
`/api/email/send` route + OAuth so the **Log as sent** button does both —
sends via Gmail API, logs locally, threads follow-ups properly. That's the
step we deferred from the original plan.

### Ramp schedule (don't skip)

| Days  | Sends/day | Notes |
|-------|-----------|-------|
| 15–17 | 5–10      | Initial sends. Read every reply within 2 hours. |
| 18–21 | 15–25     | Watch Postmaster Tools — reputation should hold green. |
| 22+   | 30–50     | Steady state. Cap at 80/day from one inbox EVER. |

At 50/day, the 219 contacts × 4-touch sequence (876 sends total) ships in
~18 sending days. Plenty of time. No need to rush.

### Daily operating rules

- **Send during the recipient's business hours**, not yours
- **Personal sender name** — "Branden Coffie", not "Mindy Outreach Team"
- **No images, no tracking pixels, no link shorteners on first touch** — all
  spam-filter triggers
- **No attachments on first touch**
- **Reply within 2 business hours** to anyone who responds — slow replies kill
  hot leads
- **Mark conversations as Important in Gmail** — improves your reputation
- **Read every bounce** — bounces above 5% will flag your domain

## Phase 6 — Operational guardrails

### Weekly check

- [ ] Open <https://postmaster.google.com> — domain reputation should be
  "Medium" or "High" (green)
- [ ] Check spam-folder placement (send one email/week to a personal Gmail
  you control, see where it lands)
- [ ] Review bounce rate in the tracker's `/outreach` → Sent tab
- [ ] Read DMARC aggregate reports (they come daily to `branden@trymindyai.com`)

### When to tighten DMARC

After **30 days of clean DMARC reports** with no unexpected senders:

- Update the DMARC TXT record at Namecheap to:
  ```
  v=DMARC1; p=quarantine; rua=mailto:branden@trymindyai.com; pct=100
  ```

After another 30 days:

- ```
  v=DMARC1; p=reject; rua=mailto:branden@trymindyai.com; pct=100
  ```

This locks in the domain so spammers can't easily impersonate you.

### If things go wrong

- **Sudden bounce spike** → stop sending immediately, check Postmaster Tools
- **Reputation drops to "Low"** → halt for 5–7 days, restart at 5/day, ramp
  again
- **Domain blacklisted** → check <https://mxtoolbox.com/blacklists.aspx>; most
  blacklists self-clear in 7–30 days if you stop sending bad traffic
- **Replies are landing in spam** → check that DKIM/SPF/DMARC all PASS on a
  test message (Phase 3 step). If they do but you're still in spam, your
  content is the problem — shorten messages, drop links, sound less
  marketer-y

## Quick-reference: what each piece does

| Piece | Where it lives | What it does |
|---|---|---|
| Workspace user | Google admin | The actual inbox `branden@trymindyai.com` lives here |
| MX record | Namecheap DNS | Tells the internet "send mail for trymindyai.com to Google" |
| SPF record | Namecheap DNS | Says "Google's servers are allowed to send as me" |
| DKIM record | Namecheap DNS | Lets Google cryptographically sign your outbound mail |
| DMARC record | Namecheap DNS | Tells receivers what to do if SPF/DKIM fail (and emails you reports) |
| Warmup tool | External SaaS | Builds positive sending reputation before cold outreach |
| The tracker | This app | Drives sequenced sends, logs every touch, surfaces who's due |
